# OpsKG 产品化对标审计与后续开发计划

> 生成日期: 2026-07-05 | 对标对象: Confluence+Rovo / BigPanda / PagerDuty / Splunk ITSI / Wiki.js

## 一、对标审计总览

### 1.1 产品定位

OpsKG 目标定位：**AI 驱动的运维知识管理与文档生成系统**，核心价值是「文档 → 知识图谱 → 智能生成」闭环。对标产品分两类：

- **知识管理类**（Confluence / Wiki.js / Outline）：侧重协作、版本、模板、搜索
- **AIOps 类**（BigPanda / PagerDuty / Splunk ITSI）：侧重图谱、事件关联、Runbook 自动化

OpsKG 的差异化在于：**两者结合 + LLM 原生**，但当前产品化程度仅约 35%。

## 六、推荐执行顺序

立即启动 Phase 1（P0），完成后再进入 Phase 2。建议按以下顺序：

1. **P0-1 文档持久化** → 解析的文件不再删除，为后续所有功能奠基
2. **P0-2 认证授权** → 所有 API 加守卫，安全第一
3. **P0-3 审计日志** → 依赖认证，记录「谁做了什么」
4. **P0-4 API测试** → 与 P0-2 并行，确保重构不破坏
5. **P0-5 备份恢复** → 最后补齐，保障数据安全
