# MarkItDown vs MinerU 真实对比评测报告

> 测试日期：2026-07-05 | 环境：Python 3.12.13(MinerU) / 3.14.4(MarkItDown) | 测试文件：4个混乱格式真实文档

|------|-----------|--------|
| 版本 | 0.0.2 | 3.4.2 |
| Python | 3.14.4 | 3.12.13 |
| 许可证 | MIT | Apache 2.0 |
| GPU需求 | 无 | 无（pipeline后端） |
| 安装方式 | `pip install markitdown` | `pip install mineru[core]` |

## 四、质量逐项对比

### 4.1 Word 文档（chaos_incident_report.docx）

原始文件包含：多级标题、粗体、列表、表格、代码块、下划线、混合字体颜色。

| 对比维度 | MarkItDown | MinerU | 优胜 |
|----------|-----------|--------|------|
| 标题层级 | ✅ `#` `##` 正确 | ✅ `#` `##` 正确 | 平局 |
| 粗体文本 | ✅ `**粗体**` 标准 | ✅ `<strong>` HTML | MarkItDown |
| 列表结构 | ✅ `*` 无序列表 | ✅ `-` 无序列表 | 平局 |
| 表格呈现 | ✅ `|--|--|` Markdown | ✅ `<table>` HTML | MarkItDown |
| 代码块 | ❌ 丢失 ` ``` ` 包裹 | ❌ 丢失 ` ``` ` 包裹 | 平局 |
| 下划线 | ❌ 丢失 | ✅ `<u>` 保留 | MinerU |
| 日期格式 | ❌ 丢失 | ✅ 保留 | MinerU |
| 可读性 | ⭐⭐⭐⭐⭐ 纯Markdown | ⭐⭐⭐ HTML混合 | MarkItDown |

**MarkItDown 输出示例**（表格部分）：
```markdown
| **步骤** | **检查项** | **结果** | **结论** |
| --- | --- | --- | --- |
| 1 | SHOW PROCESSLIST | 大量 SELECT 处于 Sending data 状态 | 存在慢查询 |
```

**MinerU 输出示例**（同一表格）：
```html
<table><tr><td><p><strong>步骤</strong></p></td><td><p><strong>检查项</strong></p></td>...
```

**结论**：MarkItDown 输出更干净、可读性更好。MinerU 保留了更多格式细节（如下划线），但 HTML 表格降低后续处理的可读性。

### 4.2 Excel 表格（chaos_server_inventory.xlsx）

原始文件包含：合并单元格（标题行 colspan）、3个Sheet、隐藏行、颜色标记。

| 对比维度 | MarkItDown | MinerU | 优胜 |
|----------|-----------|--------|------|
| 多Sheet | ✅ 3个Sheet均提取 | ✅ 3个Sheet均提取 | 平局 |
| 合并单元格 | ❌ 显示为 `Unnamed: X` | ✅ `colspan="6"` 正确 | **MinerU** |
| 隐藏行 | ❌ 仍然提取（不期望） | ❌ 仍然提取（不期望） | 平局 |
| 表格结构 | ❌ 标题行合并信息丢失 | ✅ 完整保留 | **MinerU** |
| 可读性 | ⭐⭐⭐ 有Unnamed列 | ⭐⭐⭐⭐ 结构完整 | **MinerU** |

**MarkItDown 输出**（服务器清单）：
```markdown
| 生产环境服务器清单（截止 2026-06-30） | Unnamed: 1 | Unnamed: 2 | Unnamed: 3 | Unnamed: 4 | Unnamed: 5 |
```

**MinerU 输出**（同一表格）：
```html
<th colspan="6"><p>生产环境服务器清单（截止 2026-06-30）</p></th>
```

**结论**：MinerU 在 Excel 表格处理上明显优于 MarkItDown。合并单元格的 colspan 属性被正确保留，而 MarkItDown 产生了 `Unnamed: X` 的脏数据。这是 MinerU 的核心优势场景。

### 4.3 HTML 页面（chaos_nginx_guide.html）

原始文件包含：内联CSS、DIV布局、`<pre><code>` 代码块、表格、警告框、链接。

| 对比维度 | MarkItDown | MinerU |
|----------|-----------|--------|
| 整体 | ✅ 64行/1529字符，完美 | ❌ 不支持 |

**MarkItDown 输出质量**：
- ✅ 标题 `h1`→`#`, `h2`→`##` 正确映射
- ✅ 内联CSS样式（颜色、粗体）被正确捕获为 Markdown 格式
- ✅ `<pre><code>` 代码块转为 ` ``` ` 围栏代码块
- ✅ 表格完整保留，含 `worker_processes` 等配置项
- ✅ 链接 `[Nginx 官方文档](https://...)` 正确保留
- ✅ 警告框（`<div>` 边框样式）转为 `**⚠️ 警告：**` 加粗文本

**结论**：MarkItDown 是 HTML 唯一选择，且转换质量极高。MinerU 不支持 HTML。

### 4.4 Markdown 文档（chaos_ops_scripts.md）

| 对比维度 | MarkItDown | MinerU |
|----------|-----------|--------|
| 整体 | ✅ 61行/1522字符，几乎无损 | ❌ 不支持 |

**结论**：MarkItDown 对 Markdown 文件几乎无损透传。MinerU 不支持 Markdown 格式。

## 六、最终推荐策略

```
┌─────────────────────────────────────────────────────────┐
│                    文档解析路由策略                        │
├──────────────┬──────────────────┬───────────────────────┤
│ 文件格式      │ 主解析器          │ 说明                  │
├──────────────┼──────────────────┼───────────────────────┤
│ .docx        │ MarkItDown       │ 速度快，可读性好        │
│ .xlsx        │ MinerU           │ 合并单元格正确处理       │
│ .pptx        │ MarkItDown       │ Office 文档统一处理     │
│ .html/.htm   │ MarkItDown       │ 完美转换，唯一选择       │
│ .md          │ 自研             │ 几乎无损透传            │
│ .pdf         │ MinerU           │ PDF 专长               │
│ .csv/.json   │ MarkItDown       │ 结构化数据              │
│ .sql         │ 自研             │ DDL/DML 分类           │
│ .txt         │ 自研             │ 简单分段               │
│ .rst/.xml等  │ Unstructured     │ 兜底后备               │
└──────────────┴──────────────────┴───────────────────────┘
```

### 核心原则

1. **MarkItDown 为主力**（覆盖 14+ 格式）：速度快、质量高、部署简单、MIT 协议
2. **MinerU 为 PDF/Excel 专项**：仅用于需要精确表格结构（合并单元格）的场景
3. **自研解析器**：处理 SQL、Markdown、TXT 等专用格式
4. **Unstructured 为兜底**：处理 rst、xml、odt、msg 等边缘格式

### MarkItDown 的不足与对策

| 不足 | 影响 | 对策 |
|------|------|------|
| 合并单元格→Unnamed:X | Excel 表格信息丢失 | 路由到 MinerU |
| 代码块无语言标记 | 代码高亮信息丢失 | 后处理：根据内容推断语言 |
| 下划线格式丢失 | 少量格式信息丢失 | 可接受，下划线在文档中非关键 |

### MinerU 的不足与对策

| 不足 | 影响 | 对策 |
|------|------|------|
| 不支持 HTML/Markdown | 格式覆盖有限 | MarkItDown 兜底 |
| 速度慢 200 倍 | 批量处理不可用 | 仅用于 PDF 和复杂 Excel |
| 输出混入 HTML | 下游处理复杂度增加 | 需 HTML→MD 后处理 |
| Python 3.14 不兼容 | 部署限制 | 使用 pyenv/docker 隔离 |
